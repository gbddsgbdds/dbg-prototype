## 背景

道诡修仙录是一个DBG（卡牌构建游戏），玩家需要选择路线、管理资源、击败Boss。当前游戏直接进入战斗，没有地图系统，缺乏策略深度和路线选择。

## 目标

实现一个可视化的地图系统，让玩家可以：
1. 查看当前层的地图节点
2. 选择不同的路线前进
3. 遇到不同类型的节点（战斗、篝火、商店、事件、Boss）
4. 清晰了解游戏进度

## 范围

### 本次包含
- 地图节点数据结构和生成逻辑
- 地图可视化组件 `MapScreen`
- 节点类型：战斗(common/elite)、篝火、商店、事件、Boss
- 玩家位置和路线选择
- 整合到游戏流程（开始游戏 → 地图 → 战斗 → 地图 → ...）
- 地图界面UI和动画

### 本次不包含
- 存档系统（另做）
- 第2/3层卡池和敌人
- 商店详细内容（先做占位）
- 事件详细内容（先做占位）
- 篝火详细内容（先做基础功能）

## 验收标准

- [ ] 开始游戏后显示地图界面
- [ ] 地图有多层结构（至少3层 + Boss层）
- [ ] 每层有分支路线可选
- [ ] 点击节点可进入对应节点类型
- [ ] 战斗节点进入战斗
- [ ] 篝火节点恢复理智/HP
- [ ] 商店节点显示占位界面（后续实现）
- [ ] 事件节点显示占位界面（后续实现）
- [ ] Boss节点进入Boss战
- [ ] 节点完成后返回地图
- [ ] 当前节点高亮显示
- [ ] 已完成节点标记
- [ ] `tsc --noEmit && vite build` 通过

## 开发计划

### 涉及文件
- `src/game/types.ts` - 新增地图相关类型
- `src/game/store.ts` - 新增地图状态和流程控制
- `src/components/MapScreen.tsx` - 新建地图组件
- `src/App.tsx` - 整合地图流程
- `src/App.css` - 地图样式
- `src/data/cards.ts` - 可能需要新增精英敌人

### 实现步骤

#### Step 1: 类型定义 (types.ts)
- MapNodeType: 节点类型枚举
- MapNode: 地图节点数据
- MapLayer: 地图层数据
- GameState 新增: currentLayer, currentNode, map, mapPath

#### Step 2: 地图生成 (store.ts)
- generateMap(): 生成地图数据
- 节点分布算法：战斗70%、篝火10%、事件10%、商店5%、精英5%
- 分支路线生成

#### Step 3: 地图组件 (MapScreen.tsx)
- 渲染地图节点
- 显示玩家当前位置
- 路线选择交互
- 节点完成状态

#### Step 4: 流程整合 (App.tsx, store.ts)
- 开始游戏 → 显示地图
- 选择节点 → 进入对应场景
- 场景完成 → 返回地图
- Boss战结束 → 胜利

#### Step 5: 样式美化 (App.css)
- 地图布局
- 节点样式
- 路线连接线
- 动画效果

#### Step 6: 验证
- tsc + build
- 游戏流程测试
