# 响应式布局优化 - 超宽屏幕适配

## 背景
游戏在超宽屏幕（>1400px）上可能存在内容过度拉伸的问题，影响游戏体验。需要为游戏容器添加最大宽度限制，确保在各种屏幕尺寸下都有良好的视觉效果。

## 目标
1. 为游戏主容器添加最大宽度限制（1200px）
2. 居中显示游戏内容
3. 为各种覆盖层添加适当的宽度限制
4. 保持小屏幕适配不受影响

## 范围
- 主战斗界面
- 开始界面
- 地图界面
- 商店界面
- 事件界面
- 成就界面
- 牌组查看界面
- 胜利/失败界面

## 验收标准
- [ ] 在1920px宽屏幕上，游戏内容居中显示，宽度限制在1200px
- [ ] 各界面组件不变形
- [ ] 保持小屏幕适配不受影响
- [ ] `tsc --noEmit && vite build` 通过

## 开发计划

### 修改文件
1. `src/App.css` - 添加超宽屏幕媒体查询

### 实现步骤
1. 添加 `@media (min-width: 1400px)` 媒体查询
2. 为 `.game-container` 设置最大宽度和居中
3. 为 `.start-screen` 各元素设置最大宽度
4. 为各种覆盖层（panel）设置最大宽度
5. 测试验证

## 技术细节
- 使用 CSS 变量控制最大宽度，便于统一调整
- 游戏容器：`max-width: 1200px; margin: 0 auto;`
- 覆盖层：根据内容量设置不同最大宽度
