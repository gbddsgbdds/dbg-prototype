# 商店与事件系统实现

## 背景
地图系统中商店和事件节点目前仅为占位实现，需要完整开发以丰富游戏体验。

## 目标
1. 实现功能完整的商店系统
2. 实现随机事件系统
3. 与现有地图系统无缝集成

## 范围

### 商店系统
- 玩家可购买卡牌（消耗金币）
- 玩家可移除卡牌（消耗金币）
- 商店展示3-4张随机卡牌
- 卡牌有不同价格（按稀有度）
- 商店界面UI

### 事件系统
- 随机触发事件
- 事件类型：
  - 正面事件（获得金币/HP/理智）
  - 负面事件（失去金币/HP/理智）
  - 选择事件（玩家做出选择，不同后果）
- 事件界面UI

### 不包括
- 卡牌升级功能（留待后续）
- 多层商店（第2/3层卡池）

## 验收标准
- [ ] 商店节点可正常进入
- [ ] 商店显示可购买的卡牌列表
- [ ] 购买卡牌后金币扣除，卡牌加入牌组
- [ ] 移除卡牌功能可用
- [ ] 事件节点可正常进入
- [ ] 事件随机触发，显示事件描述
- [ ] 选择类事件正常工作
- [ ] 通过 `tsc --noEmit && vite build` 验证
- [ ] 游戏可正常运行

## 开发计划

### 文件改动
1. `src/game/types.ts` - 新增 ShopItem, GameEvent 类型
2. `src/data/events.ts` - 新建事件数据
3. `src/game/store.ts` - 新增商店/事件状态和方法
4. `src/components/ShopScreen.tsx` - 新建商店UI
5. `src/components/EventScreen.tsx` - 新建事件UI
6. `src/App.tsx` - 集成商店/事件界面
