# 需求文档：战斗反馈优化

## 背景
当前战斗系统已有基本的日志显示伤害信息，但缺乏视觉反馈。玩家在打出攻击卡牌或受到敌人攻击时，只能通过战斗日志看到伤害数值，缺乏直观的视觉冲击感。这导致战斗体验较为平淡，玩家难以快速感知战斗进程。

## 目标
1. 实现伤害飘字动画，在敌人/玩家受到伤害时显示浮动数字
2. 添加音效占位系统框架，便于后期接入实际音效资源
3. 提升战斗的打击感和反馈感

## 范围

### 包含
- 伤害飘字组件（支持玩家受伤、敌人受伤、治疗效果显示）
- 飘字动画效果（上升、渐隐、颜色区分）
- 音效管理器（占位实现，使用 Web Audio API 生成简单提示音）
- 触发点集成（在 playCard、endPlayerTurn 等关键位置触发）

### 不包含
- 实际音效资源文件（使用占位音）
- 复杂的粒子效果系统
- 背景音乐系统
- 移动端振动反馈

## 验收标准

### 功能验收
- [ ] 敌人受到伤害时，在敌人位置显示红色飘字动画
- [ ] 玩家受到伤害时，在玩家状态区域显示红色飘字
- [ ] 治疗效果显示绿色飘字
- [ ] 护甲获得显示蓝色飘字
- [ ] 飘字有上升+渐隐动画效果
- [ ] 多次伤害连续触发时飘字不重叠
- [ ] 音效管理器可正常调用（占位音）

### 技术验收
- [ ] `tsc --noEmit` 无错误
- [ ] `vite build` 成功
- [ ] 不影响现有游戏逻辑
- [ ] 动画流畅不卡顿

### 体验验收
- [ ] 战斗打击感明显提升
- [ ] 飘字不会遮挡重要信息
- [ ] 视觉反馈清晰易懂

## 开发计划

### 涉及文件
1. `src/components/DamageFloat.tsx` - 新建飘字组件
2. `src/utils/soundManager.ts` - 新建音效管理器
3. `src/game/store.ts` - 添加飘字状态管理
4. `src/App.tsx` - 集成飘字组件
5. `src/App.css` - 添加飘字样式

### 实现步骤

#### 步骤1：创建飘字组件（DamageFloat.tsx）
- 定义飘字数据结构（id, value, type, position）
- 实现单个飘字动画（framer-motion）
- 支持多种类型（伤害/治疗/护甲）

#### 步骤2：添加状态管理（store.ts）
- 添加 `damageFloats` 数组到 GameState
- 添加 `showDamageFloat` 方法
- 在伤害/治疗/护甲计算处调用

#### 步骤3：创建音效管理器（soundManager.ts）
- 实现 SoundManager 类
- 使用 Web Audio API 生成简单提示音
- 提供 playSound(type) 接口
- 支持音效开关

#### 步骤4：集成到战斗流程
- 修改 playCard 中的伤害计算
- 修改 endPlayerTurn 中的敌人攻击
- 添加治疗、护甲飘字

#### 步骤5：样式美化
- 飘字字体、颜色、阴影
- 动画曲线调整
- 位置计算

#### 步骤6：测试验证
- 运行 tsc 和 build
- 实际游戏测试
