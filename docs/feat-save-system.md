# 需求文档：存档系统

> 创建时间: 2026-03-17
> 状态: ✅ 已完成
> 优先级: P0

---

## 背景

当前游戏没有存档功能，玩家无法保存游戏进度。每次刷新页面或关闭浏览器都会丢失所有进度。这对玩家体验影响很大，特别是在 Boss 战斗中或完成多个节点后意外丢失进度。

## 目标

实现一个基于 localStorage 的存档系统，让玩家可以：
1. **自动保存** - 无感知的自动存档
2. **继续游戏** - 有存档时可以选择继续之前的进度
3. **稳定可靠** - 刷新页面后能恢复完整状态

## 范围

### 包含

- 自动保存机制（回合结束、战斗结束、进入地图时触发）
- 开始界面增加"继续游戏"按钮（有存档时显示）
- 存档数据验证（版本号、数据完整性）
- 清除存档功能（设置菜单或结束界面）

### 不包含

- 多存档槽（后续版本）
- 云端存档（后续版本）
- 存档导入导出（后续版本）

## 技术方案

### 实现方式

使用 Zustand 的 `persist` middleware：
- 自动持久化到 localStorage
- 支持版本迁移
- 简化代码改动

### 触发时机

自动保存在以下时机触发（通过 Zustand persist 自动处理）：
1. 任何状态变更时自动保存
2. 页面刷新时自动恢复

### UI 变更

1. **开始界面**
   - 有存档时显示"继续修仙"按钮（绿色）
   - "重新开始" / "开始修仙" 按钮（清除旧存档并开始新游戏）
