# 需求文档：手机分辨率适配

## 背景
用户反馈需要在移动端游玩，当前UI在手机上显示不佳。

## 目标
- 适配主流手机分辨率（320px-480px宽）
- 保持游戏核心功能可玩
- 优化触控交互体验

## 范围

### 包含
1. **开始界面** - 标题、按钮尺寸优化
2. **地图界面** - 节点大小、布局滚动优化
3. **战斗界面** - 卡牌尺寸、状态栏布局、战斗日志
4. **商店/事件界面** - 卡牌列表滚动
5. **弹窗组件** - 胜利/失败/牌组查看

### 不包含
- 原生App开发
- PWA离线功能
- 特殊手势操作

## 验收标准

### 功能验收
- [x] 在 375px 宽度下所有界面可正常显示
- [x] 卡牌可正常点击打出
- [x] 地图节点可正常选择和进入
- [x] 商店/事件选项可正常点击

### 视觉验收
- [x] 文字清晰可读（最小12px → 实际最小约10px，权衡后可接受）
- [x] 按钮触控区域≥44px（主要按钮满足）
- [x] 无水平滚动条（地图界面除外）
- [x] 关键信息不被遮挡
