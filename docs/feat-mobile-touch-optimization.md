# 移动端触控优化

> 创建时间: 2026-03-17 04:10
> 状态: 开发中

---

## 背景

手机分辨率适配已完成（CSS媒体查询），但触控体验仍有问题：
- 触控设备无hover效果，卡牌选择困难
- 手牌区域容易误触
- 按钮触控区域可能过小
- 地图节点触控体验不佳

## 目标

优化移动端触控体验，使游戏在触控设备上更易操作。

## 范围

### 包含
1. 为触控设备添加明显的可交互状态指示
2. 增大关键元素的触控区域
3. 添加触控反馈效果
4. 优化手牌区域的触控交互

### 不包含
- 桌面端hover效果修改
- 游戏逻辑变更
- 新功能添加

## 验收标准

- [x] 触控设备上可打出卡牌有明显视觉指示
- [x] 手牌区域触控区域足够大（≥48px）
- [x] 按钮触控区域足够大
- [x] 地图节点触控体验流畅
- [x] `tsc --noEmit && vite build` 通过
- [ ] 在移动端设备/模拟器上测试通过（待用户验证）

---

## 开发计划

### 涉及文件
1. `src/App.css` - 添加触控优化CSS
2. `src/components/Hand.tsx` - 优化卡牌触控交互
3. `src/components/MapScreen.css` - 地图节点触控优化

### 实现步骤

1. **CSS触控优化**
   - 添加 `@media (hover: none)` 媒体查询
   - 为触控设备添加明显的可交互状态
   - 增大触控区域

2. **Hand组件优化**
   - 触控设备上卡牌更明显的可打出指示
   - 添加触控激活状态样式

3. **MapScreen优化**
   - 增大地图节点触控区域
   - 优化节点选中反馈
