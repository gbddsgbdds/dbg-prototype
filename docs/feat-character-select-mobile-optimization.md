# 需求文档：角色选择界面移动端深度适配

## 背景
角色选择界面（CharacterSelect）当前只有 768px 的媒体查询断点，缺少 400px 及以下的深度适配。在小屏幕手机上，标题、角色卡片和按钮尺寸可能过大或布局不合理。

## 目标
1. 补充 400px 及以下的媒体查询
2. 优化小屏幕下的布局和字体大小
3. 保持触控友好性（按钮最小 44px 触控区域）

## 范围
**包含：**
- CharacterSelect.css 的媒体查询补充
- 小屏幕下的布局调整

**不包含：**
- 其他组件的响应式优化
- 功能性改动

## 验收标准
1. 在 375px 宽度下，角色选择界面正常显示
2. 所有文字清晰可读
3. 按钮触控区域满足 44px 最小要求
4. 角色卡片宽度适配屏幕
5. 通过 `tsc --noEmit && vite build` 验证

## 开发计划

### 涉及文件
- `src/components/CharacterSelect.css`

### 实现步骤
1. 添加 400px 及以下的媒体查询
2. 调整标题、副标题字体大小
3. 调整角色卡片内边距和字体
4. 调整角色属性网格布局
5. 调整选择按钮尺寸
6. 验证构建
